4.1.3 Evaluator Data Structures In addition to deﬁning the external syntax of expressions, the evaluator implementation must also deﬁne the data structures that the evaluator manipulates internally, as part of the execution of a program, such as the representation of procedures and environments and the representation of true and false. Testing of predicates For conditionals, we accept anything to be true that is not the explicit false object. (define (true? x) (not (eq? x false))) (define (false? x)(eq? x false)) Representing procedures To handle primitives, we assume that we have available the following procedures: • (apply-primitive-procedure ⟨proc⟩⟨args⟩) applies the given primitive procedure to the argument values in the list ⟨args ⟩ and returns the result of the application. • (primitive-procedure? ⟨proc⟩) tests whether ⟨proc ⟩ is a primitive procedure. ese mechanisms for handling primitives are further described in Sec- tion 4.1.4.
