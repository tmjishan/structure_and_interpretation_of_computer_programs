Speciﬁcally: • e evaluator enables us to deal with nested expressions. For example, although simply applying primitives would suﬃce for evaluating the expression (+ 1 6), it is not adequate for handling (+ 1 (* 2 3)). As far as the primitive procedure + is concerned, its arguments must be numbers, and it would choke if we passed it the expression (* 2 3) as an argument. One important role of the evaluator is to choreograph procedure composition so that (* 2 3) is reduced to 6 before being passed as an argument to +. • e evaluator allows us to use variables. For example, the primitive procedure for addition has no way to deal with expressions such as (+ x 1). We need an evaluator to keep track of variables and obtain their values before invoking the primitive procedures. • e evaluator allows us to deﬁne compound procedures. is involves keeping track of procedure deﬁnitions, knowing how to use these deﬁnitions in evaluating ex- pressions, and providing a mechanism that enables procedures to accept arguments. • e evaluator provides the special forms, which must be evaluated diﬀerently from procedure calls. 493
