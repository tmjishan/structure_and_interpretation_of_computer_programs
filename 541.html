As ex- plained in Section 3.2, an environment is a sequence of frames, where each frame is a table of bindings that associate variables with their cor- responding values. We use the following operations for manipulating environments: • (lookup-variable-value ⟨var⟩⟨env⟩) returns the value that is bound to the symbol ⟨var ⟩ in the envi- ronment ⟨env ⟩, or signals an error if the variable is unbound. • (extend-environment ⟨variables⟩⟨values⟩⟨base-env⟩) returns a new environment, consisting of a new frame in which the symbols in the list ⟨variables ⟩ are bound to the corresponding elements in the list ⟨values ⟩, where the enclosing environment is the environment ⟨base-env ⟩. • (define-variable! ⟨var⟩⟨value⟩⟨env⟩) adds to the ﬁrst frame in the environment ⟨env ⟩ a new binding that associates the variable ⟨var ⟩ with the value ⟨value ⟩. 513
