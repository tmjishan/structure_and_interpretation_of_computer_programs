We have not shown the part of the controller that implements primitives, but we presume that these instructions make use of primitive arithmetic operations in the machine’s data paths. Consider how much less code would be generated if the compiler could open-code primitives—that is, if it could generate code to directly use these primitive machine operations. e ex- pression (+ a 1) might be compiled into something as sim- ple as 43 (assign val (op lookup-variable-value)(const a)(reg env)) (assign val (op +) (reg val)(const 1)) In this exercise we will extend our compiler to support open coding of selected primitives. Special-purpose code will be generated for calls to these primitive procedures instead of the general procedure-application code. In order to support this, we will augment our machine with special argument registers arg1 and arg2. e primitive arithmetic opera- tions of the machine will take their inputs from arg1 and arg2. e results may be put into val, arg1, or arg2. e compiler must be able to recognize the application of an open-coded primitive in the source program.
