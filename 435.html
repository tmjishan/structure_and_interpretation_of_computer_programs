If several such changes might be made concurrently (as with two depositors accessing a joint account) we need some way to ensure that our system behaves correctly. For example, in the case of withdrawals from a joint bank ac- count, we must ensure that money is conserved. To make concurrent programs behave correctly, we may have to place some restrictions on concurrent execution. One possible restriction on concurrency would stipulate that no two operations that change any shared state variables can occur at the same time. is is an extremely stringent requirement. For distributed bank- ing, it would require the system designer to ensure that only one trans- action could proceed at a time. is would be both ineﬃcient and overly conservative.
